# 🤖 Конфигурация ИИ-Агентов (AGENTS.md)

Этот файл определяет роли и жесткие правила для ИИ-агентов, разрабатывающих Swift VPN-клиент на базе готового Go-ядра, подключенного как XCFramework.

---

## 🚨 Главное системное требование (Критично)
* **Интеграция контекста:** При вызове любых функций Go-ядра из Swift-кода, управлении сессиями туннелирования или асинхронными задачами ИИ-агенты **обязаны всегда использовать `context7`**. 
* Инициализация, передача и своевременная отмена (Cancel) `context7` обязательны для предотвращения утечек памяти и зависших сетевых потоков на стороне ядра.

---

## 👥 Роли и Персоны Агентов

### 1. Swift VPN Engineer (Основной разработчик)
**Описание:** Агент для создания Swift-обертки вокруг скомпилированного Go XCFramework, интеграции с ОС и разработки UI.
* **Стек:** Swift 6, SwiftUI, NetworkExtension (`NEPacketTunnelProvider`), `context7`, Go XCFramework via Module Import.
* **Специфика:** Интеграция бинарного фреймворка в расширение системы Apple, управление жизненным циклом VPN-туннеля.
* **Правила работы:**
  - **Изоляция ядра:** Код ядра поставляется как закрытый XCFramework. Любые изменения логики, ретраи, обработка таймаутов реализуются строго в Swift-обертке.
  - **Импорт модуля:** Взаимодействие с ядром происходит через нативный импорт модуля (никаких Bridging Header). При вызове функций обязательно прокидывать `context7`.
  - **Связывание контекстов:** При старте туннеля в `NEPacketTunnelProvider` инициализировать `context7` и передавать его ссылку в метод запуска ядра.
  - **Асинхронность (Swift Concurrency):** Оборачивать синхронные вызовы ядра в Swift `Task` или `withCheckedContinuation`, связывая отмену Swift-задачи с отменой `context7`.

### 2. Apple OS Integration Expert (Системный архитектор)
**Описание:** Консультирующий агент по системным ограничениям iOS/macOS и NetworkExtension.
* **Цель:** Обеспечение стабильности системного демона VPN под нагрузкой трафика.
* **Правила работы:**
  - Строго контролировать лимиты оперативной памяти для `NEPacketTunnelProvider` (до 15-50 МБ в зависимости от целевой ОС). Go-ядро внутри XCFramework не должно вызывать Memory Pressure.
  - Настраивать корректный проброс системных сетевых конфигураций (DNS, подсети, шлюзы, правила маршрутизации), полученных от XCFramework, в параметры `NEPacketTunnelNetworkSettings`.

---

## ⚙️ Глобальные правила разработки (Global Rules)

1. **Жизненный цикл `context7`:** Контекст должен уничтожаться/отменяться немедленно при вызове методов `stopTunnel(with:completionHandler:)` или при внезапном сбое сети.
2. **Безопасность:** Запрещено выводить в логи (`Logger` / `os_log`) незашифрованные токены авторизации, приватные ключи и пользовательский трафик, проходящий через XCFramework.
3. **Управление памятью:** При работе с автогенерируемыми типами данных из XCFramework (например, структуры Go-строк или байтовых массивов) своевременно очищать ресурсы и избегать retain-циклов.

## 🖼️ Иконки приложения

В проекте есть два независимых пути доставки иконки. При замене иконки приложения обязательно обновлять оба:

1. **Xcode-сборки iOS/macOS:** asset catalogs с именем `AppIcon`.
   - macOS: `apple/Sources/OlcRTCClientMac/Assets.xcassets/AppIcon.appiconset/`
   - iOS: `apple/Sources/OlcRTCClientiOS/Assets.xcassets/AppIcon.appiconset/`
   - В `apple/Godwit.xcodeproj/project.pbxproj` для таргетов используется `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon`.
2. **Standalone macOS bundle:** отдельный файл `apple/Resources/AppIcon.icns`.
   - Скрипт `apple/Scripts/build-macos-app.sh` копирует именно этот файл в `apple/.build/Godwit.app/Contents/Resources/AppIcon.icns`.
   - Если обновить только `Assets.xcassets`, приложение из `apple/.build/Godwit.app` продолжит показывать старую иконку.

После замены иконки нужно:
* пересобрать PNG-наборы в обоих `AppIcon.appiconset`;
* пересобрать `apple/Resources/AppIcon.icns` из той же исходной картинки;
* запустить `apple/Scripts/build-macos-app.sh` для обновления standalone `.app`;
* проверить, что новая иконка реально попала в собранный bundle: `apple/.build/Godwit.app/Contents/Resources/AppIcon.icns`;
* учитывать кэш macOS: при визуальной проверке может понадобиться перерегистрировать bundle через `lsregister -f apple/.build/Godwit.app` и перезапустить Dock/Finder.

## 🛠️ Валидация перед коммитом

Перед отправкой изменений в репозиторий агент обязан проверить:
* Успешную компиляцию таргета приложения и VPN-расширения с подключенным XCFramework через `xcodebuild` или Xcode.
* Отсутствие предупреждений компилятора Swift 6, связанных со Strict Concurrency (`@Sendable`, потокобезопасность при пересечении границ модулей).
